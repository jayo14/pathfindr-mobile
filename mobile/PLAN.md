# Build PathFindr as a bright campus navigation app

## Features

- [x] Show an interactive campus map with the user’s location, key buildings, and grouped map pins in busy areas.
- [x] Let students search for buildings, departments, and facilities with quick category filters.
- [x] Open a place details view with photo, description, category, and a button to start directions.
- [x] Draw walking directions between places with distance and estimated walking time.
- [x] Show upcoming campus events in a simple browsable feed.
- [x] Add a lost and found area where people can post lost or found items with image, description, and location.
- [x] Handle location permission clearly with helpful prompts and fallback states if access is denied.
- [x] Support a basic offline experience by saving key campus places and the last viewed map area.
- [x] Prepare the app to connect cleanly to live campus data later while shipping polished starter content now.

## Design

- [x] Use a bright, student-friendly visual style with fresh greens, soft neutrals, and high-contrast map accents.
- [x] Make the map the visual hero, with clean floating cards, rounded sheets, and mobile-native spacing.
- [x] Give search, event, and reporting flows a lively but professional feel inspired by premium navigation apps.
- [x] Use subtle motion for marker taps, screen transitions, route loading, and primary button presses.
- [x] Keep empty states and loading states friendly, clear, and visually polished instead of plain placeholders.

## Pages / Screens

- [x] **Splash**: Branded opening screen with a crisp first impression and fast transition into the app.
- [x] **Onboarding**: Short welcome flow explaining navigation, events, and lost and found, plus location access guidance.
- [x] **Map**: Main campus map with live position, place pins, quick search access, and route entry points.
- [x] **Search**: Search results screen with category chips and fast place discovery.
- [x] **Building Details**: Place profile with image, summary, and directions action.
- [x] **Directions**: Route screen showing origin, destination, route line, walking time, and distance.
- [x] **Events**: Campus event list with date, location, and quick details.
- [x] **Lost & Found**: Browse reports and create a new lost or found item submission.
- [x] **Settings**: App preferences, offline info, permissions help, and support links.

## Data & Behavior

- [x] Start with polished LASUSTECH-ready sample places, categories, events, and lost-item content so the app looks complete immediately.
- [x] Structure the app so live campus data can replace the starter content smoothly later.
- [x] Save core place data and recent map context for a reliable first offline experience.
- [x] Show clear feedback for loading, no results, permission issues, and network problems.
